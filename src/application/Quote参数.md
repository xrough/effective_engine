# 报价参数与风险参数说明

在报价参数中，首先需要定义围绕理论价格形成买卖报价的方法。系统先计算一个理论中间价，这个中间价可以来自盘口买一卖一的中点、模型公平价或流动性加权价格，然后按基点换算出基础价差，其中相对价差等于基点数除以一万，半价差等于中间价乘以相对价差再除以二，买价等于中间价减去半价差，卖价等于中间价加上半价差；如果需要避免价格很高时价差绝对值过大，也可以在此基础上加入最小跳动单位约束。

基础挂单数量表示在没有库存压力和风险限制时系统愿意提供的流动性规模，可以通过目标名义金额除以价格与合约乘数得到数量，也可以直接用历史成交统计得到一个稳定区间，然后再根据市场状态做缩放。随着交易进行系统会形成净仓，因此需要库存偏斜规则来移动报价，常见做法是用库存规模或组合 Delta 乘以一个价格偏移系数得到偏移量，再把偏移量加到中间价上重新生成买卖价，并对偏移量设定上下界防止过度移动。

市场波动升高时，报价会按当前波动与基准波动的比值放大基础价差，例如用滚动收益率标准差计算当前波动，再与长期中位数比较得到比例，按比例放大价差以减少 adverse selection 风险。为了避免行情变化时旧报价被吃掉，还需要设置报价最大存活时间，当当前时间减去最后一次报价时间超过阈值就刷新报价，同时设置撤单触发阈值，当新理论价格与当前挂单价格的相对差值换算成基点超过阈值才撤单重挂，从而在减少撤单成本与避免错误价格之间取得平衡。

在风险参数中，需要定义系统绝不允许突破的安全边界。单笔交易上限通过数量与名义金额双重限制实现，其中名义金额等于数量乘以价格再乘以合约乘数，若计划交易超过阈值则拒绝或裁剪到最大允许数量。仓位限制通过净仓与毛仓控制，净仓限制要求当前仓位加计划交易后绝对值不超过阈值，毛仓限制则统计多仓绝对值与空仓绝对值之和避免复杂对冲导致仓位失控。组合风险上限通过汇总所有持仓的名义金额或 Delta 等价规模计算，例如把每个头寸的 Delta 乘以数量与合约乘数求和得到组合 Delta，并在交易前做前瞻检查。

亏损与回撤限制通过成交记录计算已实现盈亏并累加得到当日盈亏，同时维护日内权益峰值并计算当前回撤，当亏损或回撤超过阈值触发熔断或降级模式。风险触发后系统进入冷却期，在设定时间内拒绝新交易以防止连续错误操作，同时还需要检查行情有效性，通过当前时间与最近行情更新时间的差值判断数据是否过旧，若超过阈值则禁止报价或下单。这些风险参数通常通过历史极端行情与策略行为统计得到，并在回测中验证在这些限制下系统仍能稳定运行，实盘中报价引擎先根据报价参数生成交易意图，再由风险管理模块依据风险参数进行最终裁剪或拒绝，从而保证收益优化与风险控制之间有清晰边界并具备可审计性。
